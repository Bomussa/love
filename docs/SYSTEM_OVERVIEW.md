# MMC‑MMS — نظرة عامة مبسطة (لغير التقنيين)

هذا المستند يشرح النظام بلغة بسيطة بدون أكواد.

## ما هو النظام؟
- نظام لتنظيم طوابير المرضى في المركز الطبي.
- واجهة للمراجع (المريض) + لوحة للإدارة.
- خادم خلفي يقدّم البيانات عبر عنوان موحّد: `/api/v1/*`.

## كيف يعمل بسرعة؟
1) المراجع يختار الخدمة (مثلاً: المختبر).
2) النظام يعطيه رقم دور ويعرض موقعه في الطابور.
3) عند اقتراب دوره تصله إشعارات فورية.
4) بعد الانتهاء، يفتح له النظام الخطوة التالية تلقائيًا (إن وجدت).

## المكوّنات الرئيسية
- الواجهة (Frontend): تطبيق ويب خفيف وماسرع (React + Vite).
- الخادم (Backend): وظائف سحابية (Supabase Functions) تقدّم البيانات.
- قناة التحديثات الحية: سيرفر‑سِنت إيفِنتس (SSE) لعرض التحديثات فورًا.

## لماذا عنوان `/api/v1/` مهم؟
- كل تواصل بين الواجهة والخادم يمر عبره.
- هذا يجعل الصيانة أسهل ويمنع الأخطاء.

## خمس ميزات أساسية
- إدارة أرقام الدور للطوابير.
- إشعارات فورية عند اقتراب الدور.
- نظام PIN لفتح العيادات.
- تقارير يومية/أسبوعية/شهرية.
- مسارات طبية ديناميكية حسب نوع الفحص.

## كيف أشغّله محليًا (للتجربة)؟
- نفتح الموقع المحلي ثم نطلب حالة النظام:
  1) تشغيل التطبيق (يحتاج مطوّر لتشغيله).
  2) في المتصفّح نكتب: `/api/v1/status` ونشاهد “200 OK” وبيانات الحالة.

## ماذا أفعل عند مشكلة؟
- إن لم تظهر بيانات الحالة:
  - تأكد أنّ العنوان يبدأ بـ `/api/v1/`.
  - حدّث الصفحة.
  - استشر مسؤول التقنية ليتأكد من اتصال الخادم.
